# プロジェクト仕様

最終確認日: 2026-07-15

## 確定: 目的

自動車販売店スタッフが、お客様役との音声またはテキスト会話を通じて応対を練習し、会話後に採点と改善点を確認できるWebアプリとする。

## 確定: 現在の対象シナリオ

1. 12カ月点検・来店促進
2. 車検誘致・電話フォロー

現在のコードに車両販売シナリオは存在しない。車両販売編は`docs/SALES_FLOW.md`で未確定として管理する。

## 確定: 現在の提供機能

- お客様発話の表示と音声再生
- スタッフの音声認識またはテキスト入力
- 状態遷移とキーワード判定による会話進行
- シナリオ別の採点、改善点、推奨トーク
- A4縦・白黒を前提とした会話と評価の印刷
- Google管理者ログインによるシナリオの下書き保存と公開
- Firestore公開データが取得できない場合のローカルデータ利用

## 確定: 技術構成

- 静的HTML、CSS、JavaScript
- GitHub Pages
- Firebase Authentication
- Cloud Firestore
- Web Speech API
- MP3音声およびブラウザー音声合成

OpenAI APIや外部LLMは現在使用していない。お客様役と採点はルールベースである。

## 確定: データの基準

- ローカル標準シナリオ: `scenario.js`
- 音声ID、音声文、MP3対応: `audio-db.js`
- 公開中シナリオ: Firestore `roleplay/public`
- 管理中の下書き: Firestore `roleplay/draft`

## 未確定

- 車両販売編の追加
- 会話ログの永続保存と保存期間
- 合格点の設定
- 長時間無音時の自動確認
- Firebase管理画面からの音声ファイル更新
- 今後のシナリオ数と権限区分
