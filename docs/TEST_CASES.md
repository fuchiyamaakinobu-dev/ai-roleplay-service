# テストケース

最終確認日: 2026-07-15

## 自動テストとして固定すべき基準

### 共通

- シナリオの配点合計が100点
- 表示文と音声登録文が一致
- `ready`音声ファイルが存在し、極端に小さくない
- 四国めたん・剣崎雌雄の車検誘致音声が存在

### 12カ月点検

- 追加整備未確認でも減点して会話継続
- 作業時間と追加整備確認を同じ発話で認識
- 「調子の悪いところや気になる点はありませんか」だけでも、お客様が「オイル交換もお願いします」と返答する
- 車両状態だけを質問し、点検以外のご用命を確認していない場合は確認項目を満点扱いにしない
- 「オイル交換もお願いします」と回答
- 再確認時は「そのほかは大丈夫です」と回答
- 作業時間を説明済みなら同じ質問を繰り返さない
- 引取理由が仕事なら土日・時間帯提案に反応
- 日付だけなら時間帯を聞き返す
- 日付と時間がそろうと予約確定
- 11時と15時のように複数の時刻を提示した場合、お客様が「早いほうでお願いします」と返答してから予約確定
- 「早いほうでお願いします」、説明の食い違い解消後、家族相談後の再連絡、近い店舗・家族来店への同意、汎用の会話継続で登録済みMP3を再生
- 「午前」「午後」だけでは具体的な時刻未確定として会話継続
- 引取を即時確定すると減点
- 引取理由の回答として家族相談をランダム表示しない
- 仕事理由では不要な他店舗提案なしでも満点にできる
- 距離・運転不安では不要な土日提案を満点条件にしない

### 車検誘致

- ロープレ開始ボタンを押すと、お客様発話を待たず直ちに音声認識がONになる
- 「もしもし」だけでは本人確認ステップを進めない
- 「お客様でしょうか」だけでは本人確認を達成しない
- 「トヨタです」だけでは店舗・担当者名を達成しない
- 名乗りを先に行った場合は本人確認を聞き返す
- 本人確認と名乗りをまとめて話しても認識
- 名乗りと「いつもお世話になっております」をまとめて話すと、減点条件を保ったまま車検案内へ進む
- 日頃の利用へのお礼なしでも減点して進行
- お礼がない場合は「こちらこそ」ではなく「はい」と返す
- 車検案内に「車検」が含まれていても、満了日と作業可能日の具体的な2つの日付がなければ時間説明へ進めない
- 車検案内と都合確認を同じ発話で行った場合、お客様は「いつできますか」と返し、満了日・作業可能日の案内を待つ
- 作業時間だけ説明済みの場合は店内待ちだけを聞き返す
- 3日前連絡だけ説明済みの場合は連絡先だけを聞き返す
- 作業時間の説明後、店内待ちの補足質問へ「もちろん。できます。」と答えると、説明を合算して代車確認へ進む
- 3日前連絡の説明後、連絡先の補足質問へ「はい」と答えると、説明を合算して予約復唱へ進む
- 「少々お待ちください」を店内待ち確認として加点しない
- 質問ではない「調子は良いですね」を車両状態確認として加点しない
- 入庫日時未確定の終話で「いつ行けばいいんですか？」
- 入庫日時確定後は任意案内を省略可能
- 省略発話中に予約復唱があれば加点
- 模範17発話で100点・正常終話

## 手動ブラウザ確認

- お客様音声が途中で切れない
- 音声終了後にマイクが開始する
- スタッフ入力欄が前回発話のまま残らない
- 短い無音でロープレ自体が終了しない
- 発話候補が変化する
- 会話分岐が事情に合う
- 採点が実際の応対と一致する
- A4縦・白黒で会話と評価が印刷される
- 管理画面の下書きが公開画面へ即時反映されない
- 「公開する」後に公開画面へ反映される
