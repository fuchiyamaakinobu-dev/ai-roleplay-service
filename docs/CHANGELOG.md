# 変更履歴

## 2026-07-24（シナリオ間の音声混在防止）

- お客様音声を同時に1件だけ再生するよう管理を追加
- シナリオ切替、やり直し、開始、手動終了、スタッフ発話送信、次音声再生時に古い音声を停止
- 停止された古い音声の終了処理から、別シナリオでマイクが遅れて開始しないよう再生世代を管理
- 車検誘致の最終お客様音声や即時引取検出音声は、正常終了時に最後まで再生する動作を維持
- 音声停止、重複再生防止、古い終了処理の無効化を確認する回帰テストを追加

## 2026-07-24（複数時刻選択の文字・音声一致）

- 会話表示、音声登録文、仕様書、回帰テストの文章を同期

## 2026-07-24（12カ月点検・追加お客様音声）

- 「では、早いほうでお願いします」の音声を追加し、複数時刻から早い時刻を選ぶ返答へ登録
- 説明の食い違い解消後、家族相談後の再連絡、近い店舗・家族来店への同意、汎用の会話継続の音声を追加
- 説明の食い違い解消後と家族相談後の再連絡について、再添付された2音声をそれぞれ正しい用途へ登録

## 2026-07-24（複数時刻候補の予約確定修正）

- 12カ月点検で11時・15時など複数の時刻候補を提示した場合、時間確定として即時終了しないよう修正
- AIお客様が「では、早いほうでお願いします」と返答し、候補中の最も早い時刻を選択してから予約確定
- 「午前」「午後」など具体的な時刻のない時間帯だけでは終話しないよう判定を強化
- 画像と同じ日時確認を回帰テストへ追加

## 2026-07-18（車検案内から時間説明への誤進行修正）

- Firestore公開シナリオで作業可能日の必須語が不足していても、具体的な2つの日付なしでは完了扱いにしない安全判定を追加
- 車検案内と都合確認をまとめて話した場合は「いつできますか」と応答し、満了日・作業可能日の説明を待つように修正
- 都合確認は質問文の場合だけ達成するよう誤判定を抑制
- 画像と同じ複合発話および公開データの不足設定を回帰テストへ追加

## 2026-07-18（車両状態確認へのお客様返答）

- 12カ月点検で「調子の悪いところや気になる点はありませんか」と確認した場合、お客様が無反応にならず「オイル交換もお願いします」と返答するよう修正
- 車両状態だけの確認と、点検以外のご用命を含む完全な確認を分けて判定し、採点条件を維持
- 画像と同じスタッフ発話を回帰テストへ追加

## 2026-07-16（分割発話時の会話ループ修正）

- 車検誘致で、作業時間と店内待ち可否を別々に回答しても両方の内容を合算して判定
- AIお客様の補足質問に対する「はい」「もちろん。できます。」などの肯定回答を質問の文脈に沿って判定
- 事前連絡日と連絡先を別々に回答しても、同じ確認を繰り返さず次のステップへ進行

## 2026-07-16（自然発話の矛盾修正）

- 本人確認、名乗り、店内待ち、車両状態確認の誤加点を抑制
- 12カ月点検の満点条件をお客様の理由別に調整
