# 変更履歴

## 2026-07-18（車検案内から時間説明への誤進行修正）

- Firestore公開シナリオで作業可能日の必須語が不足していても、具体的な2つの日付なしでは完了扱いにしない安全判定を追加
- 車検案内と都合確認をまとめて話した場合は「いつできますか」と応答し、満了日・作業可能日の説明を待つように修正
- 都合確認は質問文の場合だけ達成するよう誤判定を抑制
- 画像と同じ複合発話および公開データの不足設定を回帰テストへ追加

## 2026-07-18（車両状態確認へのお客様返答）

- 12カ月点検で「調子の悪いところや気になる点はありませんか」と確認した場合、お客様が無反応にならず「オイル交換もお願いします」と返答するよう修正
- 車両状態だけの確認と、点検以外のご用命を含む完全な確認を分けて判定し、採点条件を維持
- 画像と同じスタッフ発話を回帰テストへ追加

## 2026-07-16（分割発話時の会話ループ修正）

- 車検誘致で、作業時間と店内待ち可否を別々に回答しても両方の内容を合算して判定
- AIお客様の補足質問に対する「はい」「もちろん。できます。」などの肯定回答を質問の文脈に沿って判定
- 事前連絡日と連絡先を別々に回答しても、同じ確認を繰り返さず次のステップへ進行

## 2026-07-16（自然発話の矛盾修正）

- 引取理由として家族相談がランダム表示される矛盾を修正
- お礼なしの場合のお客様応答を「はい。」へ修正
- 作業時間説明済み・3日前連絡説明済みの場合は不足項目だけを聞き返すよう修正
- 本人確認、名乗り、店内待ち、車両状態確認の誤加点を抑制
- 12カ月点検の満点条件をお客様の理由別に調整
- 新しい聞き返し2件を四国めたん・剣崎雌雄で追加

## 2026-07-16（スタッフ開始時のマイク）

- スタッフ発話から始まる車検誘致で、ロープレ開始直後に音声認識を自動開始
- 開始時に前回のスタッフ入力欄を消去
- お客様発話から始まるシナリオは従来どおり音声終了後に開始

## 2026-07-16（名乗り・お礼の同時発話）

- 店舗・担当者名の名乗りと日頃の挨拶を同じ発話で処理できるよう修正
- 必須キーワード不足時はお礼項目を加点せず、会話のみ車検案内へ進める仕様を維持
- 次の車検案内後に「こちらこそ。」が遅れて表示される会話ずれを解消
